7.5 基数排序

受《啊哈算法》启发，先看一个简单的例子，桶排序（bucket sort）：

将要排序的数看成是另一个数组 temp 的下标 i，temp[i] 代表此数的出现的次数；遍历 temp 时，输出 temp[i] 个 i。

当待排序的数范围很小分布均匀且是正整数时，可以试试，挺有意思。

// 对范围在 [0, 9] 之间的整数排序
int[] arr = {0,1,7,9,9,4,4,4,4,3,5,6};
int[] temp = new int[10];
int n = arr.length;
int m = temp.length;

// n 次比较
for (int i = 0; i < n; i++) {
    // 将 arr[i] 看成是 temp 的下标
    temp[arr[i]]++;
}   
/*
此时 temp：{1, 1, 0, 1, 4, 1, 1, 1, 0, 2}
代表 0 出现了 1 次
1 出现了 1 次
2 出现了 0 次
3 出现了 1 次
4 出现了 4 次
5 出现了 1 次
6 出现了 1 次
7 出现了 1 次
8 出现了 0 次
9 出现了 2 次
次数总和为 n
*/
// m 次比较
for(int i = 0; i < m; i++) {
    // n 次比较（？）
    for (int j = 0; j < temp[i]; j++) {
        // 可以在外部定义变量 index = 0，改为 arr[index++] = i
        System.out.print(i + " ");
    }
}
System.out.println();

一共 T(2n + m）次比较，时间复杂度为 O(n+m）；当 m < n 时，可以记为 O(n）。

缺点是如果数组中有一个特别大的数 max，m 也会特别大（m = max + 1），会造成 temp 中大部分元素值都是空（即 0），遍历 temp 时花费时间将会更多；空间复杂度为 O(m），这种版本的桶排序无法保存对象，只能记录相同 key 的对象的个数，讨论是不是稳定排序没有意义（如果可以保存，则是稳定排序），桶排序的完整代码在最后面。

？处的解释：temp：{1, 1, 0, 1, 4, 1, 1, 1, 0, 2}；

j = 0 时，j < temp[0]，遍历 1 次；j < temp[1] 时，遍历 1 次；j < temp[2] 时，遍历 0 次，... j < temp[9] 时，遍历 2 次，总共 n 次。（temp[i] 记录了 i 在 arr 中出现的次数，总和为 n）

i = 0，i < m，其实是 m + 1 次比较：0 < m、1 < m、2 < m、... m - 1 < m、 m < m 为 fasle，退出循环，为了简化计算，忽略了。

基数排序（Radix Sort）算法思想：按元素的各个位的值，将元素分配到不同的 “桶” 中；从个位数开始，到最高位结束。

以 {6，56，5，125，84，2，9，58，981} 为例：

第一轮排序取元素的个位数：6，6，5，5，4，2，9，8，1

把元素的个位数当作下标，个位数决定了元素在 temp 中的位置：

temp[6][0] = 6，temp[6][1] = 56
temp[5][0] = 5，temp[5][1] = 125
temp[4][0] = 84
temp[2][0] = 2
temp[9][0] = 9
temp[8][0] = 58
temp[1][0] = 981

设元素的个位数为 digit，因为有多个 digit 为同一个值的元素，为了避免元素被覆盖，使用二维数组保存这些元素；同时需要定义一个 count 数组记录个位数为 digit 的元素的个数；例：count[6] 默认为 0，temp[6][count[6]] = temp[6][0] = 6，随后 count[6]++ 变为 1，表示个位数为 6 的元素有 1 个，再 temp[6][count[6]] = temp[6][1] = 56，count[6]++ 变为 2，表示个位数为 6 的元素有 2 个...

此时的 temp 为：

{
    {},
    {981},
    {2},
    {},
    {84},
    {5, 125},
    {6, 56},
    {},
    {58},
    {9}
}

将其放回到原来的数组 arr；因为 count[i] 代表个位数为 i 的元素有 count[i] 个，如 count[0] = 0，代表 temp[0] 中一个元素都没有；count[5] = 2，代表 temp[5] 中有两个元素：temp[5][0]、temp[5][1]。

当 count[i] 不等于 0 时才需要将元素放入 arr。

for (int i = 0, index = 0; i < count.length, i++) {
    // count[5] = 2，进入
    if (count[i] != 0) {
        // j < 2 时，j = 0、1
        // 得到 temp[5][0]、temp[5][1] 赋给 arr[index]
        for (int j = 0; j < count[i]; j++) {
            arr[index++] = temp[i][j];
        }
        // 清空 count[i] 以便保存下一轮 temp[i] 元素的个数
        count[i] = 0;
    }
}

此时 arr 为：{981，2，84，5，125，6，56，58，9}。

按顺序取出元素的十位数：8，0，8，0，2，0，5，5，0

temp[8][0] = 981
temp[0][0] = 2
temp[8][1] = 84
temp[0][1] = 5, temp[2][0] = 125
temp[0][2] = 6, temp[5][0] = 56
temp[5][1] = 58
temp[0][3] = 9

此时 temp 为：

{
    {2, 5, 6, 9},
    {},
    {125},
    {},
    {},
    {56, 58},
    {},
    {},
    {981, 84}
}

此时 arr 为：{2，5，6，9，125，56，58，981，84}。

按顺序取出元素的百位数（最大元素的最高位）：0，0，0，0，1，0，0，9，0

temp[0][0] = 2, temp[0][1] = 5, temp[0][2] = 6, temp[0][3] = 9
temp[1][0] = 125
temp[0][4] = 56, temp[0][5] = 58
temp[9][0] = 981, temp[0][6] = 84

此时 temp 为：

{
    {2, 5, 6, 9, 56, 58, 84},// 百位数都为 0
    {125},
    {},
    {},
    {},
    {},
    {},
    {},
    {},
    {981}
}

至此 2，5，6，9，56，58，84，125，981 有序。

因为某位上的值的范围在 [0, 9] 内，所以可以定义 temp 为一个长度为 10 的二维数组，但一维数组的长度无法确定，先考虑最差情况，所有元素的某位数上的值都相同时，需要长度为 n 的一维数组才能装下。

int[] arr = {6, 56, 5, 125, 84, 2, 9, 58, 981};
int n = arr.length;
int[][] temp = new int[10][n];
// 保存某位数相等的元素的个数
int[] count = new int[10];

for (int j = 0; j < n; j++) {
    int num = arr[j];
    // 取元素的个位数
    int digit = num % 10;
    temp[digit][count[digit]++] = num;
}
// 将 temp 中的元素放入 arr
for (int i = 0, index = 0; i < count.length; i++) {
    if (count[i] != 0) {
        for (int j = 0; j < count[i]; j++) {
            arr[index++] = temp[i][j];
        }
    }
    count[i] = 0;
}

for (int j = 0; j < n; j++) {
    int num = arr[j];
    // 取元素的十位数
    int digit = num / 10 % 10;
    temp[digit][count[digit]++] = num;
}
for (int i = 0, index = 0; i < count.length; i++) {
    if (count[i] != 0) {
        for (int j = 0; j < count[i]; j++) {
            arr[index++] = temp[i][j];
        }
    }
    count[i] = 0;
}

for (int j = 0; j < n; j++) {
    int num = arr[j];
    // 取元素的百位数
    int digit = num / 100 % 10;
    temp[digit][count[digit]++] = num;
}
for (int i = 0, index = 0; i < count.length; i++) {
    if (count[i] != 0) {
        for (int j = 0; j < count[i]; j++) {
            arr[index++] = temp[i][j];
        }
    }
    count[i] = 0;
}

System.out.println(Arrays.toString(arr));

因为我们事先知道最大数的最高位是百位，所以三轮结束。否则就需要找到数组中的最大值 max，来确定轮数。

不妨让 max = 101，一共三轮；记 k = 1，每次乘等 10，第三轮时 max / 100 = 1，第四轮 max / 1000 = 0，所以只要满足 max / k > 0 就继续循环。

// 使用前，先确定数组长度不是 0
public static int findMax(int[] arr) {
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if(arr[i] > max) {
            max = arr[i];
        }
    }
    return max;
}
public static void radixSort(int[] arr) {
    int n = arr.length;
    // temp 是长度为 10 的二维数组
    int[][] temp = new int[10][n];
    int[] count = new int[10];

    // n 次，暂时忽略不计
    int max = findMax(arr);

    // 轮数与最大值的位数有关，记为 p，比较 p 次
    for (int k = 1; max / k > 0; k *= 10) {
        // 比较 n 次
        for (int j = 0; j < n; j++) {
            int num = arr[j];
            int digit = num / k % 10;
            temp[digit][count[digit]++] = num;
        }
        // 比较 10 次
        for (int i = 0, index = 0; i < count.length; i++) {
            // 比较 n 次，之前？分析过，类似，不再赘述
            for (int j = 0; j < count[i]; j++) {
                arr[index++] = temp[i][j];
            }
            count[i] = 0;
        }
    }
}

记 temp 的长度为 k，则 count 的长度也应为 k，count[i] 记录着 temp[i] 数组中元素的实际个数。

从比较次数上看，T(n）= p*(k+2n）+ p，记为 O( p(k+n）），这个版本的 k 是确定的，为 10，即 O(pn）。

从空间上看，S(n）= kn+k，记为 O(kn），是稳定排序。

与桶排序一样，基数排序不能对负数排序。当 k 太大时，没有足够的空间存放 temp 数组，会报 OutOfMemoryError 堆空间不足错误。（实例都在堆中分配空间）

桶排序的完整代码：

public static void bucketSort(int[] arr) {
    int max = findMax(arr);
    // 创建长度为 max + 1 的数组
    int[] temp = new int[max + 1];
    int n = arr.length;
    int m = temp.length;

    for (int i = 0; i < n; i++) {
        temp[arr[i]]++;
    } 
    int index = 0;
    for(int i = 0; i < m; i++) {
        for (int j = 0; j < temp[i]; j++) {
            arr[index++] = i;
        }
    }
}

如果想保存对象：

当然可以使用集合代替，自动扩容，可以节省很大部分空间，我将测试代码放在了 test 目录的 com.cqh.sort.radix 下。

桶排序的优化

如果数组中的元素在 [0，max] 内分布均匀，如 {9，2，3，1，5，6，3，4，8}，使用桶排序没有太大问题；但待排序的数组是 {9993，9996，9999，9998，9998，9997}，创建长度为 10000 的 temp 数组，太浪费空间。

就以上面的数组为例：

找出最小值：9993，最大值：9999，创建长度为 9999 - 9993 + 1 = 7 的 temp 数组。

将要排序的数 num 看成是数组 temp 的下标 num - min，temp[i] 代表 i + min 的出现的次数；遍历 temp 时，输出 temp[i] 个 i + min。

temp：{1，0，0，1，1，2，1} 分别代表 9993、9994、... 9999 出现的次数。

public static void bucketSort(int[] arr) {
    int n = arr.length;
	int max = arr[0];
    int min = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
        if (arr[i] < min) {
            min = arr[i];
        }
    }
    
    int[] temp = new int[max - min + 1];
    
    for (int i = 0; i < n; i++) {
        temp[arr[i] - min]++;
    } 
    int index = 0;
    for(int i = 0; i < temp.length; i++) {
        for (int j = 0; j < temp[i]; j++) {
            arr[index++] = i + min;
        }
    }
}

计数排序 counting sort

temp：{1，0，0，1，1，2，1}

由于 temp 里的元素总和为 n，不妨让 temp[i] = temp[i] + temp[i - 1]，得到它们实际的排名。

temp：{1，1，1，2，3，5，6}，如 9993 排第 1 个，9996 是第 2 个，9997 第 3，9998 第 4、5，9999 第 6，存入时将其减一。

新创建一个数组 b，遍历 arr，于是有 b[--temp[arr[i] - min]] = arr[i]；arr[i] - min 得到 arr[i] 对应的排名在 temp 数组中的下标；--temp[arr[i] - min] 即是 arr[i] 对应的排名。

以 9998 为例，将其代入，b[--temp[9998 - 9993]] = b[--temp[5]]，先减减，temp[5] = temp[5] - 1 = 4；b[4] = 9998，此 9998 排第 5；然后又有一个 9998，将其代入 b[--temp[5]]，由于之前 temp[5] 减减变为 4，这次有减减变为 3，所以 b[3] = 9998，此 9998 排第 4，这样两个 9998 在 b 中就排好序了。

int n = arr.length;
int[] b = new int[n];
int max = arr[0];
int min = arr[0];
b[0] = arr[0];
for (int i = 1; i < n; i++) {
    if (arr[i] > max) {
        max = arr[i];
    }
    if (arr[i] < min) {
        min = arr[i];
    }
    b[i] = arr[i];
}
int m = max - min + 1;
int[] temp = new int[m];

for (int i = 0; i < n; i++) {
    temp[arr[i] - min]++;
} 
for (int i = 1; i < m; i++) {
    temp[i] = temp[i] + temp[i - 1];
}
// 因为是对 arr 排序，所以将 arr、b 互换了
for(int i = 0; i < n; i++) {
    arr[--temp[b[i] - min]] = b[i];
}

T(n）= 5n + m，S(n）= n + m；
