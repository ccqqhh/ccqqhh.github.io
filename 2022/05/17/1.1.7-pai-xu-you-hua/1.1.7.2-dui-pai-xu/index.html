7.2 堆排序

内容导视：

  • 树
  • 二叉树
  • 堆排序

由于堆排序需要使用到树这种数据结构，所以简单过一下。

7.2.1 树

树（Tree）是 n 个结点的集合，n = 0 时称为空树；若不是空树，则树 T 由被称作根 R 的结点以及 0 个或多个互不相交的非空子树 T1、T2、...、Tk 组成。

每棵子树的根被称作根 R 的子结点，而 R 是每棵子树的根的父结点。

由于 T1、T2...它们也是树，同样也是由根结点与若干个子树组成，举例：

A 的子结点有 D、E、F，G 的父结点有 C；那些没有子结点的结点，如 D、E、F、B...称为树叶结点，也称叶子结点；有子结点的结点称为分支结点，除根结点 R 外，分支结点也被称为内部结点。

树的层次：从根 R 为第一层开始算，R 的子结点 A、B、C 为第二层，A、B、C 的子结点 D、E、F、G 为第三层；最大层次称为树的高度。

7.2.2 二叉树

二叉树是一颗树，树中的每个结点最多只能有两个子结点，称为左结点、右结点。（每个结点的子结点有 0 或 1 或 2 个）

满二叉树的所有树叶结点都在最后一层，所有的分支结点都有两个子结点，此时结点总个数 = 2k - 1。（k 为树的高度）如图：

此满二叉树的结点个数为 24 - 1 = 15；同样高度的二叉树中，满二叉树的结点个数最多，树叶结点个数最多，每一层都是满的。

完全二叉树

对树中的 n 个结点按从上至下、从左到右的顺序进行编号，如果编号为 i（1 i n）的结点与同样高度的满二叉树中编号为 i 的结点在二叉树中的位置相同，则这棵二叉树称为完全二叉树，如下：

从图像上看，必须从左至右将每层铺满，才能继续在下一层放置结点；如下就不是完全二叉树：

第四层并没有从最左边开始放置，前面还空了三个位置，不是完全二叉树。

二叉树性质

  1. 在第 i 层最多有 2 个结点。（1、2、4、8、16...）

  2. 高度为 k 的二叉树最多有 2k - 1 个结点。（1、3、7、15、31...）

  3. 高度为 k 的完全二叉树至少有 个结点。（1、2、4、8...）

    证明：最少结点的完全二叉树就是（k - 1）高度的满二叉树的结点个数 + 1 = 。

  4. 树叶结点个数为 n0，有两个子树的结点个数为 n2，有 n0 = n2 + 1。

    证明：设一共 n 个结点，每个结点都通过一条线连接着它的父结点，除了根结点，则一共有 n - 1 条线。

    定义 n1 为有一个子树的结点个数，有 n - 1 = n1 + 2 * n2；（一个结点有两个子树代表两条线，有一个子树代表一条线，把它们都加起来）

    因为二叉树结点只包括这三种类型，所以有 n = n0 + n1 + n2，

    n - 1 = n1 + 2 * n2 = n0 + n1 + n2 - 1，整理得到 n2 = n0 - 1。

  5. 具有 n 个结点的完全二叉树的高度为 ⌊log2n⌋ + 1。

    证明：设完全二叉树的高度为 k，n 一定小于（k + 1）高度的完全二叉树的结点个数；根据性质 3 得：，两边取对数：，整理得：，又因 k 是整数，当 log2n 为整数时，k = log2n + 1；不为整数时，k = ⌊log2n⌋ + 1。

    综合 k = ⌊log2n⌋ + 1。

    ⌊⌋ 是向下取整（floor），⌈⌉ 是向上取整（ceiling），如 ⌊3.5⌋ = 3。

  6. 一共有 n 个结点的完全二叉树，序号从 1 开始，则最后一个内部结点的序号为 n / 2。

    证明：首先最后一个叶子结点的父结点为最后一个内部结点。

    观察规律，对任意结点 i，它的左结点序号应为 2i，右结点序号应为 2i + 1。最后一个叶子结点序号为 n，它的父结点序号为 ⌊n / 2⌋。

7.2.3 堆排序

Heap Sort 是对简单选择排序的改进。

之前的选择排序并没有将每轮的比较结果保存，后几轮的比较其实前面已经做过了；你可以回看之前的数组中的简单排序的那个例子，如第 2 轮中又重新拿 2 与其它数比较了一次；简单选择排序最好最差时间复杂度都是 O(n2）。

而堆天然具有这种优势。

堆是具有以下性质的二叉树：

  • 每个结点的值都大于等于其左右子结点的值，称为大顶堆（根结点为最大值）
  • 每个结点的值都小于等于其左右子结点的值，称为小顶堆（根结点为最小值）

heap sort 算法思想：将待排序的数组构造成一个大顶堆，此时根结点为最大值，让根结点与树中的最后一个结点交换位置（删去根结点）；剩下的 n - 1 个元素重新构造成一个大顶堆，取出第二大值，与倒数第二个元素交换位置（删去根结点）... 如此反复操作，得到有序数组。

不构造大顶堆（不利用堆保存比较结果的性质）

1、如何调整使得根结点为最大值？

调整：从最后一个分支结点开始，使它与自己的子结点比较、交换，保证此分支结点为最大值；然后继续往前找其它的分支结点 ... 重复以上操作，直到根结点为最大值；

将根结点与最后一个结点交换，剩下的 n - 1 个结点继续调整，找出第二大值，与倒数第二个结点交换 ... 直至整个数组有序。

2、如何得到最后一个分支结点的下标？

因为数组下标 i 从 0 开始，所以根据之前的二叉树性质 6，再加上 int 类型可以略去小数，做了一些修改：（将长度为 n 的数组看作成一个完全二叉树）

对任意分支结点 i，它的左结点应为 2 * i + 1，右结点序号应为 2 * i + 2。最后一个叶子结点下标为 n - 1，它的父结点即最后一个分支结点的下标为 n / 2 - 1。

在这个图中，也就是拿 arr[6] 与它的子结点比较、交换，使得 arr[6] 为这三个结点中的最大值；arr[5]、arr[4] ... 同样如此；直到 arr[0] 与 arr[1]、arr[2] 比较、交换成为最大值后，将 arr[0] 与最后一个元素 arr[14] 交换，则末尾元素为最大值；

删除根结点后，将剩下的 14 个元素看作一个完全二叉树，再次调整，找出此树的最大值（相当于整个数组的第二大值），并与 arr[13] 交换 ... 剩下的 2 个元素中找出大值，与 arr[1] 交换，至此数组有序。

一根连线就相当于一次比较，第一轮中 15 个元素为一颗树，14 次比较；第二轮 14 个元素为一颗树，13 次比较 ... 第十四轮 2 个元素，1 次比较，很明显时间复杂度为 O(n2）；

但是需要考虑到最后一个分支结点没有右结点，即 2 * i + 1 = n - 1 时，访问 arr[2 * i + 2] 会导致越界，此时不能进行左右结点的比较。

public static void selectSort(int[] arr) {
    // 总轮数为 n - 1
    for (int n = arr.length; n > 1; n--) {
        // 从最后一个分支结点开始进行比较、交换，直到 arr[0] 为树中的最大值
        for (int i = n / 2 - 1; i >= 0; i--) {
            adjust(arr, i, n);
        }
        // 将最大值与树中的最后一个结点交换位置
        swap(arr, 0, n - 1);
    }
}
public static void adjust(int[] arr, int i, int n) {
    // 获取分支结点的左右结点下标
    int leftChild = leftChild(i);
    int rightChild = leftChild + 1;
	// 指针默认指向左结点
    int pointer = leftChild;
    
    // 代表只有左结点，没有右结点，不用进行左右结点的比较
    if (leftChild == n - 1) {
        
	// 两个结点都有
    } else {
        // 若左结点小于右结点，指针指向右结点，保证了指针指向的结点为大值
        if (arr[leftChild] < arr[rightChild]) {
            pointer++;
        }  
    }
    // 若分支结点小于大值结点，则交换它们位置，保证分支结点最大
    if (arr[i] < arr[pointer]) {
        swap(arr, i, pointer);
    }
}
// 获得 i 对应的左结点下标
public static int leftChild(int i) {
    return 2 * i + 1;
}
public static void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}

上面这种方式并没有调整成大顶堆，以 {1，2，3，4，5} 为例：

从最后一个分支结点开始，arr[5 / 2 - 1] = arr[1] = 2，与 arr[3]、arr[4] 比较后，arr[1] 与 arr[4] 交换位置：

往前找分支结点即 arr[0]，与 arr[1]、arr[2] 比较后，arr[0] 与 arr[1] 交换位置：

此时根结点 arr[0] 为最大值，很明显 1 与 5 交换后，1 小于 4 和 2，不是大顶堆；应该拿被替换的 1 继续与它的两个子结点比较，让 1、4 替换，我们之前的代码没有这一步。

也就是说 arr[i] 不仅要与 arr[2 * i + 1]、arr[2 * i + 2] 比较，在 arr[i] 与其中一个结点交换后（记 arr[i] 交换后的下标为 pointer），arr[pointer] 应继续往下与它的子结点比较。（别越界）

在没有交换的情况下，以上图为例，arr[0] > arr[1、2]，由于 arr[1] > arr[3、4]，所以肯定有 arr[0] > arr[3、4]，不用继续往下比较。

/**
 * 以 arr[i] 为根节点的二叉树，经过调整后，使得 arr[i] 为这个二叉树的最大值
 *
 * @param arr 待调整的数组
 * @param i 开始调整的位置
 * @param n 二叉树结点的个数
 */
public static void adjust(int[] arr, int i, int n) {
    /*
    	pointer 默认指向 i 的左结点。当 arr[i]、arr[pointer] 交换后，arr[pointer] 应该继
    	续与它的左结点 arr[2 * pointer + 1]、arr[2 * pointer + 2] 比较，在其期间保证别越
    	界（小于 n）。
    */
    for (int pointer = leftChild(i); pointer < n; pointer = leftChild(i)) {
        int leftChild = pointer;
        int rightChild = leftChild + 1;

        if (leftChild < n - 1) {
            if (arr[leftChild] < arr[rightChild]) {
                pointer++;
            }
        } 

        if (arr[i] < arr[pointer]) {
            swap(arr, i, pointer);
            // i 指向被交换的元素，此时 pointer 应为 leftChild(i)，arr[i] 继续与它的子结点比较
            i = pointer;
            
        // 若 arr[i] > arr[pointer]，没有交换，没有必要继续往下比较 
        } else {
            break;
        }
    }
}

构建大顶堆的好处

以此图为例：

构建大顶堆后，将根结点与末尾结点交换位置，剩下的 n - 1 个元素成一颗树：

再重新构建大顶堆，此时就不用那么繁琐，而是从根结点 arr[0] 开始与左右子结点比较：

arr[0] 与 arr[1]、arr[2] 比较
arr[0] 与 arr[1] 交换，即 8、14 交换
arr[1] 与 arr[3]、arr[4] 比较
arr[1] 与 arr[3] 交换，即 8、9 交换

这个调整过程正好对应修改后的 adjust 方法所做的操作。

可以看到只有被交换后的结点无法保证是三个结点中的最大值，所以才需要继续往下与它的左右结点比较；那么还有另一边的结点，如 arr[2]，由于大顶堆的性质，再加上它一直没被动过，arr[2] > arr[5] 这是确定的，不用再次比较。

每次构建成大顶堆后，只需将根结点与末尾结点交换（每次都取出大顶堆中的最大值放在后面），重复操作，直至数组有序。

代码如下：

public static void heapSort(int[] arr) {
    int n = arr.length;
    // 构建大顶堆
    for (int i = n / 2 - 1; i >= 0; i--) {
        adjust(arr, i, n);
    }
    /*
    	先将根结点与末尾结点交换，剩下的 n - 1 个元素重新构建成大顶堆（n > 1）
    */
    for (; n > 1; n--) {
        swap(arr, 0, n - 1);
        adjust(arr, 0, n - 1);
    }
    // 上面的循环可以替换为如下代码
    /*
    	for (n--; n > 0; n--) {
            swap(arr, 0, n);
            adjust(arr, 0, n);
        }
    */
}

除了第一轮使用 O(n）次比较构建成了大顶堆，之后的 n - 2 轮每次只比较了一半的结点，为 O（log2n），总时间复杂度为 O(nlogkn），是不稳定排序。

另外提一句：

adjust 方法中循环条件 pointer < n 除了控制防止越界，还有其它讲究。

当 n = 2 时，因为在上一次循环中，调用 adjust 方法使得只剩两个结点的二叉树中的 arr[0] 为最大值（被删除的结点不算）；在这次循环中， swap(arr，0，1），交换两个结点位置，使得 arr[0] 为最小值，接着调用 adjust(arr，0，1），在此方法中的 n = 1，只有一个结点的二叉树没有必要调整，应退出循环。

但是 adjust 并没有加上 n > 1 作为循环条件，因为 i = 0 时，pointer = leftChild(0）= 1，pointer < n，1 < 1，不满足条件直接退出了循环，结束了 adjust 方法的执行；再回到这里，执行 n--，n = 1，不满足条件，退出循环，至此排序结束。

你如果学过了面向对象，就知道 heapSort 与 adjust 方法中的 n 变量分别是两块独立的内存空间，互不影响。
